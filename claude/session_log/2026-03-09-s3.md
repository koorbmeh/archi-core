## Design Decisions

- Provider dispatch uses thin functions, not a class hierarchy. Archi can
  refactor this when it develops model selection strategy.
- Errors don't crash — return ModelResponse with error field. This lets the
  generation loop decide how to handle failures without try/except everywhere.
- Error responses don't accrue cost (no tokens consumed).
- Cost table is a module constant, not config. Archi can move it to
  capability_registry metadata when it needs to manage costs dynamically.
- Budget check happens before the call (pre-check), not after. Once you've
  hit the ceiling, no more calls. Post-call warning logged if the call that
  just completed pushed total past the limit.
