## Design Decisions

- capability_registry uses JSON (not Markdown) — machine-native, Archi reads
  and writes it programmatically. Resolves the open question from session 1.
- gap_detector has one hardcoded thing: KERNEL_COMPONENTS — the kernel's own
  expected wiring. Everything else emerges from evidence. This is the minimum
  bootstrap knowledge needed.
- Operational gaps scale priority with failure frequency (0.7 base + 0.05 per
  instance, capped at 1.0). Repeated failures surface faster.
- Deduplication prefers the "higher" source (structural > registry > operational)
  and merges evidence from all sources.
