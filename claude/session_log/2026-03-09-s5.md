## Capability Built

**Constraint enforcement** (`src/kernel/alignment_gates.py`) — kernel-level
invariants that survive self-modification. Every action passes through
declarative gates that enforce budget ceilings, protected file integrity,
external action transparency, and scope boundaries.

## Design Decisions

- Gates are pure functions: each takes ActionContext, returns GateResult. No side
  effects, no shared state. Easy to test, easy to compose.
- Budget gate checks three tiers in order: session → daily → monthly. Short-circuits
  at the first violation.
- Scope gate uses metadata["source"] to distinguish kernel code from generated code.
  Generated code cannot write to src/kernel/ — only the kernel itself can.
- PROTECTED_FILES duplicated in both alignment_gates.py and self_modifier.py. This
  is intentional defense in depth — even if one check is somehow bypassed, the other
  catches it.
- cost_log.jsonl is separate from operation_log.jsonl. Budget tracking is an alignment
  concern, not an operational concern.
