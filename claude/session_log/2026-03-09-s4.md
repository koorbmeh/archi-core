# Session 4 — 2026-03-09

## Capability Built

**Self-development cycle** (`src/kernel/generation_loop.py`) — the unified
loop that wires all kernel modules into Observe → Detect Gap → Plan →
Generate Code → Test → Integrate. When this runs end-to-end against a live
model, Archi can develop itself.

## What Was Done

- Built `src/kernel/generation_loop.py` (199 lines)
  - `run_cycle(repo_path, registry, log_path) → CycleResult` — single public function
  - Phase 1 (Observe): calls `detect_gaps()` from gap_detector, picks highest-priority gap
  - Phase 2 (Plan): asks model to produce a JSON plan (file_path, description, approach, deps)
  - Phase 3 (Generate): asks model to write the complete Python source file
  - Phase 4 (Test + Integrate): passes generated code to `apply_change()` from self_modifier
  - On success: registers new capability via capability_registry
  - Every phase logs structured entries to `data/operation_log.jsonl` for feedback
  - Operation log entries feed back into gap_detector's operational gap detection
  - Injectable `plan_fn` and `generate_fn` for testing without API keys
  - Handles: budget exceeded, model errors, parse failures, short code, test failures
  - Strips markdown fences from model output (plan JSON and generated code)
- Wrote 19 unit tests across 7 test classes
  - Plan parsing: valid JSON, markdown fences, invalid JSON, missing keys, defaults
  - Operation logging: writes JSONL, appends, missing_capability field
  - No gaps: cycle stops at observe
  - Plan failures: model error, parse failure, budget exceeded
  - Generate failures: model error, too short, budget exceeded, fence stripping
  - Full integration: success (real git repo, tests pass, capability registered)
  - Full integration: failure (tests fail, rollback, no registration)
  - Feedback loop: failed operation log entries create operational gaps
- Total: 74/74 tests passing (55 existing + 19 new)

## Design Decisions

- Single `run_cycle()` function, not a class. The loop is stateless — all state
  lives in the registry, the git repo, and the operation log. No object needed.
- `plan_fn` and `generate_fn` are injectable callables. Tests mock them without
  patching. Archi can swap in different models per phase when it develops
  model selection strategy.
- Plan is JSON with required keys (file_path, description, approach). Minimal
  structure — enough for the loop to act, flexible enough for Archi to extend.
- Operation log uses the same JSONL format gap_detector already reads. This
  creates a feedback loop: failed cycles produce operational gaps that
  future cycles can detect and address.
- Code minimum length (20 chars) catches empty/trivial model responses before
  attempting the expensive git branch + test cycle.
